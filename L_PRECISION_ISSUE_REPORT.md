# L 题近共线精度问题处理报告

## 1. 问题结论

用户指出的说法是成立的，并且暴露了 L 题当前配置中的真实问题。

严格来说，这不是题意本身的数学定义错误，也不是 checker 的误差判定公式错误；问题出在两处：

1. 官方标准解在计算两个方向之间的极小夹角时，使用了两个极角直接相减：

```cpp
width = angle_b - angle_a;
```

当两个点方向非常接近，且坐标较大时，`atan2` 得到的两个极角本身约为同一个中等大小的数，相减后只剩一个极小差值。这个过程会发生明显的浮点消元误差。

2. 原有数据没有覆盖这种“近共线大坐标”的反例，所以错误做法可以通过全部数据。

因此，用户提交的反例说明：当前题目数据强度不足，并且旧标准解存在数值稳定性问题。需要修复标准解并加强数据。

## 2. 反例说明

反例输入为：

```text
1
2 2
1000000 999999
999999 999998
```

两个点分别为：

```text
A = (1000000, 999999)
B = (999999, 999998)
```

它们的叉积为：

```text
cross = 1000000 * 999998 - 999999 * 999999 = -1
```

按从 `B` 到 `A` 的逆时针小夹角计算时，绝对叉积为 `1`，点积为：

```text
dot = 1000000 * 999999 + 999999 * 999998 = 1999996000002
```

所以正确夹角为：

```text
atan(1 / 1999996000002)
```

最远点距离平方为：

```text
1000000^2 + 999999^2 = 1999998000001
```

正确面积为：

```text
1/2 * 1999998000001 * atan(1 / 1999996000002)
```

小数约为：

```text
0.500000500000750...
```

如果程序输出：

```text
0.500000473783
```

绝对误差约为：

```text
2.62e-8
```

这已经大于题目要求的 `1e-9` 绝对/相对误差，因此这种输出不应被判为正确。

## 3. 错误原因

旧标准解对每个方向记录极角：

```cpp
long double ang = atan2l(y, x);
```

之后滑动窗口计算角度宽度时，用的是：

```cpp
long double width = angle_at(r - 1) - angle_at(l);
```

对于普通数据，这样通常能得到足够精度。但在本反例中，两个方向非常接近，真实夹角约为：

```text
5e-13
```

而两个极角本身约为：

```text
0.785398...
```

用两个接近的浮点数相减，会丢失大量有效数字。最终面积会被放大到明显错误的程度。

这个错误不是 checker 放宽造成的。当前 checker 的判定逻辑是：

```cpp
tol = 1e-9 * max(1.0, abs(answer))
```

对于本反例，容许误差约为 `1e-9`。错误输出 `0.500000473783` 会被 checker 正确拒绝。之前能通过，是因为数据集中没有这个反例。

## 4. 修复方式

修复核心是：不要用两个极角相减求夹角，而是用两个方向向量的叉积和点积直接求夹角。

对于两个方向向量 `u = (ux, uy)` 和 `v = (vx, vy)`，逆时针夹角可以用：

```cpp
cross = ux * vy - uy * vx;
dot = ux * vx + uy * vy;
angle = atan2(cross, dot);
if (angle < 0) angle += 2 * pi;
```

这种写法的优势是：

1. 对近共线小角度更稳定。
2. 不需要先得到两个接近的极角再相减。
3. `atan2(cross, dot)` 正好表达两个向量之间的有向夹角。

为避免整数乘法溢出，代码中使用 `__int128` 计算叉积和点积：

```cpp
__int128 cross = (__int128)u.dx * v.dy - (__int128)u.dy * v.dx;
__int128 dot = (__int128)u.dx * v.dx + (__int128)u.dy * v.dy;
long double res = atan2l((long double)cross, (long double)dot);
if (res < 0) res += 2 * PI;
```

同时，每个方向组除了记录极角、最大半径平方、数量之外，还记录约分后的方向向量：

```cpp
struct Group {
    long double ang;
    long double r2;
    long long dx;
    long long dy;
    int cnt;
};
```

极角仍然用于排序，真正计算窗口宽度时改用 `atan2(cross, dot)`。

## 7. 最终判断

用户反馈代表题目当前版本确实存在需要修复的问题。

问题性质是：

```text
题意无误，checker 误差口径无误；
旧标准解数值稳定性不足，原数据未覆盖该边界。
```

修复后：

1. 标准解改为数值稳定的 `atan2(cross, dot)` 夹角计算。
2. 数据集中加入近共线大坐标反例。
3. 构建脚本和审计脚本同步更新，避免后续重新生成时遗漏该问题。

因此，该问题已经被修复，并且修复后的数据能够有效阻止直接用两个极角相减计算夹角的错误提交通过。
