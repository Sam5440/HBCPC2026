## 1. B 题：固定输出

这题没有真正的算法。程序不需要读取输入，只要原样输出题目要求的祝贺语句即可。

做法：

1. 直接 `cout` 固定字符串。
2. 注意换行。

复杂度：时间 `O(1)`，空间 `O(1)`。

## 2. I 题：山海关老冰糕

题意可以理解为：有 `N` 条生产线，第 `i` 条生产线每 `t_i` 分钟生产 1 份冰糕。问最少多少分钟可以生产至少 `M` 份。

关键观察：

如果给定一个时间 `T`，第 `i` 条生产线能生产 `floor(T / t_i)` 份，所以总产量可以直接算出来。时间越长，产量越多，因此答案具有单调性，可以二分。

做法：

1. 设二分下界为 `0`。
2. 设上界为最快生产线单独生产 `M` 份所需时间，即 `min(t_i) * M`。
3. 对中间时间 `mid` 计算总产量。
4. 若总产量不少于 `M`，说明时间够用，缩小右边界。
5. 否则时间不够，增大左边界。

复杂度：时间 `O(N log(min(t_i) * M))`，空间 `O(N)`。

## 3. H 题：可能是字符串签到题

题目问区间 `s[l,r]` 中出现次数最多的子串出现了多少次。虽然题面说“子串”，但任意非空子串的出现次数都不会超过它第一个字符的出现次数。因此最优答案一定可以由某个单字符子串取得。

关键观察：

在一个区间内，出现次数最多的子串，答案等价于出现次数最多的字符。

做法：

1. 预处理 `pref[i][c]`，表示前 `i` 个字符中字符 `c` 出现了多少次。
2. 每次查询 `[l,r]` 时，字符 `c` 的出现次数为 `pref[r][c] - pref[l-1][c]`。
3. 枚举 26 个小写字母，取最大值。

复杂度：预处理 `O(26n)`，单次查询 `O(26)`，空间 `O(26n)`。

## 4. J 题：海鲜大排档点生蚝

每个摊位有单买价格 `A` 和整打价格 `B`，一打是 12 个。只能选择一个摊位，求预算 `M` 内最多能买多少个。

关键观察：

如果 `B >= 12A`，买一打并不比买 12 个单个便宜，直接全买单个最优。

如果 `B < 12A`，整打更划算，最优策略一定接近“尽量多买整打”。由于余钱可能刚好买几个单个，所以只需要比较：

1. 买最多的整打数 `d = floor(M / B)`。
2. 少买一打，即 `d - 1`，把省下的钱用于单买。

做法：

1. 对每个摊位计算该摊位下的最大购买数量。
2. 若 `B >= 12A`，答案是 `M / A`。
3. 否则比较 `d` 打和 `d-1` 打两种情况。
4. 维护最大数量；若并列，保留编号更小的摊位。

复杂度：时间 `O(N)`，空间 `O(1)`。

## 5. F 题：猫 01 序列

这是一道交互题。隐藏序列是长度为 `n` 的 01 序列，目标是求 1 的总数 `S`。一次询问选择一个集合 `I`，评测机返回：

`x * (S - x)`

其中 `x` 是集合 `I` 内 1 的个数。

关键观察：

如果我们猜总数是 `S`，那么每次返回值 `p` 必须满足：

`p = x(S-x)`

等价于二次方程有整数解：

`S^2 - 4p` 必须是完全平方数，并且根的奇偶性要正确。

做法：

1. 对二进制位 `0..16` 各询问一次。
2. 第 `b` 次询问所有编号第 `b` 位为 1 的位置。
3. 收集 17 个返回值。
4. 枚举候选答案 `S=1..n`。
5. 对每个返回值检查 `S^2 - 4p` 是否为完全平方数。
6. 第一个满足全部检查的 `S` 就是答案。

复杂度：询问次数固定 17 次，本地计算 `O(17n)`，空间 `O(1)`。

## 6. A 题：求求你不要再摔薯片了

每包薯片有 1 到 5 的味道。处理每包薯片时，可以选择保留在前半段，或扔到最后。最终顺序就是“没扔的子序列”接上“扔到最后的子序列”，两个子序列内部都保持原相对顺序。

关键观察：

每次新加入一包薯片，只会影响它所在子序列的最后一个味道。因为味道只有 5 种，所以可以用很小的状态记录两个子序列的首尾味道。

做法：

1. 把最终序列看成两个保序子序列 `keep` 和 `move` 的拼接。
2. 动态规划状态记录：
   `keep` 的首味道、尾味道，`move` 的首味道、尾味道。
3. 处理当前味道 `v` 时，有两种选择：
   放进 `keep`，更新 `keep` 尾味道和收益。
   放进 `move`，更新 `move` 尾味道和收益。
4. 所有薯片处理完后，如果两个子序列都非空，还要加上 `keep` 尾到 `move` 首的拼接收益。
5. 最后再加上每包薯片本身提供的 `n` 点基础快乐值。

状态数是 `6^4`，其中 0 表示某个子序列还为空。

复杂度：时间 `O(n * 6^4)`，空间 `O(6^4)`。

## 7. K 题：灯带

要构造一个长度为 `n` 的 RGB 灯带，红绿蓝数量分别固定为 `r,g,b`。给出若干互不相交的区间，每个区间内最多只能出现两种颜色。

关键观察：

因为限制区间两两不相交，所以可以逐个处理区间，不会出现“一个位置同时受多个区间约束”的麻烦。

做法：

1. 对每个限制区间，选择一种颜色不在该区间使用。
2. 被允许的另外两种颜色必须还够填满这个区间。
3. 参考实现优先避开剩余数量较少的颜色，尽量消耗数量较多的颜色。
4. 区间内填完后，剩余不受限制的位置用剩余颜色补齐。
5. 最后检查 RGB 数量是否精确满足，且每个限制区间颜色种类不超过 2。

复杂度：时间 `O(n + m)`，空间 `O(n)`。

## 8. E 题：电梯

需要构造尽量少的电梯，使任意两层楼 `x,y` 都能由某一部电梯直接到达。每部电梯的停靠层是一个严格递增序列，且必须停 1 层和 `n` 层。

关键观察：

任意楼层对 `(l,r)` 都可以看成一个需要被覆盖的区间。只要某部电梯的停靠层中同时包含 `l` 和 `r`，这个楼层对就被覆盖了。

做法：

1. 枚举所有楼层对 `(l,r)`，把它看成区间 `[l,r]`。
2. 用类似“区间分组”的贪心方法，把这些区间分到若干组。
3. 对每个新区间，优先复用已经结束的组；如果没有可复用的组，就新建一组。
4. 每一组对应一部电梯。
5. 输出该组内所有区间端点，再补上 1 和 `n`。

参考实现最终得到的电梯数量为：

`floor(n/2) * ceil(n/2)`

复杂度：枚举所有楼层对，时间 `O(n^2 log n)`，空间 `O(n^2)`。由于 `n <= 1000`，可接受。

## 9. L 题：贪吃蛇

蛇从原点出发，选择一个角度区间，然后张开扇形并向外扩展，吃掉区间内的食物。要至少吃掉 `k` 个食物，使扇形面积最小。

关键观察：

一个扇形面积为：

`1/2 * 角度宽度 * 半径^2`

如果选定了一组食物，最小角度区间就是这组食物极角的最小覆盖区间，半径就是其中最远食物到原点的距离。

做法：

1. 把每个食物转换为极角 `angle` 和距离平方 `r^2`。
2. 按极角排序。
3. 为处理跨越 `0` 角度的情况，把数组复制一份，第二份角度加 `2π`。
4. 滑动窗口枚举连续的 `k` 个食物。
5. 窗口角度宽度是右端角度减左端角度。
6. 窗口内最大 `r^2` 用单调队列维护。
7. 取所有窗口面积的最小值。

复杂度：排序 `O(n log n)`，滑动窗口 `O(n)`，空间 `O(n)`。

## 10. M 题：拯救猫猫

网格中有障碍、起点 `S`、终点 `E` 和若干朝固定方向看的狗。猫不能走到障碍、狗所在格，也不能进入任何狗的视线。可以让一只狗睡着，睡着的狗失去视线但仍不能被经过。

关键观察：

判断某只狗睡着后是否可行，可以直接模拟剩余狗的视线，然后从 `S` 做 BFS 看能否到达 `E`。

做法：

1. 扫描网格，记录所有狗的位置和方向，同时记录 `S` 与 `E`。
2. 写一个函数 `can(sleep)`，表示编号为 `sleep` 的狗睡着时是否存在路径。
3. 在 `can` 中，从每只未睡着的狗出发，沿视线方向标记被看见的格子，直到遇到障碍或另一只狗。
4. 把障碍、狗、被看见的格子都视为不可走。
5. 从 `S` BFS，能到 `E` 则可行。
6. 若不需要任何狗睡着也可达，则输出字典序最小的狗。
7. 否则按行列顺序枚举每只狗，输出第一个可行的狗；若都不可行，输出 `-1 -1`。

复杂度：设狗数量为 `d`，网格大小为 `nm`，直接实现约为 `O(d * nm)` 级别。当前参考实现采用直接模拟，便于理解和验证。

## 11. C 题：世界树

给定带权树，多次查询 `x,y`，求：

`max_u(dis(x,u) + dis(y,u))`

关键观察：

树上任意两点距离唯一。难点在于查询次数和点数都可能很大。当前参考实现按数据结构分成几类处理。

做法：

1. 小规模树：
   对每个点做一次 DFS，预处理任意两点距离。查询时枚举 `u` 取最大值。
2. 链形树：
   把每个点的位置转成前缀距离 `pref[i]`。
   对查询 `x,y`，最优点 `u` 只可能落在三类位置：`x,y` 之间、二者左侧、二者右侧。
   用前缀最小值和后缀最大值即可 `O(1)` 回答。
3. 星形树：
   最优点只可能是中心、`x`、`y`，或权值最大的若干叶子。
   参考实现保留前 5 个最大叶子作为候选，逐个计算。
4. 其他情况：
   参考实现直接对每次查询分别从 `x` 和 `y` 做 DFS，再枚举所有点。

复杂度：

小树预处理约 `O(n^2)`；链查询 `O(1)`；星查询 `O(1)`；一般直接处理为单次 `O(n)`。该实现与本项目生成数据的结构相匹配。

## 12. G 题：建设高铁

图上每条边有修建难度。若购买价格为 `V` 的设备，就能修所有难度不超过 `V` 的边。每次查询要选择当前繁荣度最高的 `k` 个城市，问要让它们连通的最小 `V`。

关键观察：

如果边按难度从小到大加入，当这 `k` 个城市第一次处于同一个连通块时，当前边的难度就是答案。这就是 Kruskal/最小瓶颈连通的思想。

做法：

1. 先把所有边按难度升序排序。
2. 遇到修改操作，就更新城市繁荣度。
3. 遇到查询操作：
   找出当前繁荣度最高的 `k` 个城市，繁荣度相同取编号小者。
   用并查集从小到大加入边。
   每个连通块维护其中被选中城市的数量。
   当某个连通块包含全部 `k` 个目标城市时，当前边权就是答案。
4. 若 `k=1`，不需要任何边，答案为 0。

复杂度：

参考实现中，一次普通查询会重新选 top-k 并重新跑并查集，复杂度约为 `O(n + m α(n))`。大规模压力数据中 `k=1` 的情况可直接 `O(1)` 回答。

## 13. D 题：简单字符串题

要统计字符串 `S` 写成：

`S = A^i B_1 B_2 ... B_k C^j`

的方案数。`A`、`C` 是重复段，`B_1...B_k` 是中间任意非空段。

关键观察：

可以先决定左边重复段总长度 `p`，右边重复段总长度 `q`。中间剩余长度是 `n-p-q`。

对于固定的 `p`：

`S` 的前 `p` 个字符能写成多少种 `A^i`？

这等价于：前缀长度 `p` 有多少个合法循环节长度。

同理，对于固定的 `q`：

`S` 的后 `q` 个字符能写成多少种 `C^j`？

做法：

1. 用 Z 函数计算每个前缀长度 `p` 的重复表示数量 `pref[p]`。
2. 把字符串反转，同样计算每个后缀长度 `q` 的重复表示数量 `suf[q]`。
3. 直接枚举 `p,q` 是 `O(n^2)`，太慢。
4. 用 NTT 做卷积，得到：
   `h[t] = sum(pref[p] * suf[q])`，其中 `p + q = t`。
5. 如果 `k=0`，中间必须为空，答案就是 `h[n]`。
6. 如果 `k>0`，中间长度为 `n-t`，要切成 `k` 个非空字符串，方案数是：
   `C(n-t-1, k-1)`
7. 累加：
   `sum h[t] * C(n-t-1, k-1)`。

为什么需要 Z 函数：

对一个候选循环节长度 `len`，如果从位置 `len` 开始能和字符串开头匹配足够长，就说明若干个 `len` 的倍数长度都可以由同一个循环节重复得到。

为什么需要 NTT：

前缀选择和后缀选择本质上要按 `p+q` 合并计数，正好是卷积。`998244353` 是常用 NTT 模数，可以高效完成多项式乘法。

复杂度：Z 函数与循环节统计约 `O(n log n)`，NTT `O(n log n)`，组合数预处理 `O(n)`，总复杂度 `O(n log n)`。
